# دليل رفع الموقع على استضافة مجانية

هذا الموقع تطبيق React/Vite ثابت (Static SPA). بعد البناء يصبح مجلد `dist` هو
كل ما تحتاجه للرفع. أفضل الاستضافات المجانية له هي **Netlify** و **Vercel**
و **Cloudflare Pages** (جميعها مجانية للمشاريع الشخصية وتعطيك رابط HTTPS فوراً).

> تم تجهيز المشروع مسبقاً: ملف `public/_redirects` و `vercel.json` يضمنان عمل
> روابط الصفحات الداخلية عند التحديث (لأن الموقع يستخدم توجيه React Router).

---

## أولاً: بناء الموقع (خطوة مشتركة)

```bash
npm install      # أول مرة فقط
npm run build    # ينتج مجلد dist الجاهز للرفع
```

بعد نجاح الأمر، يظهر مجلد `dist/` ويحتوي على: `index.html` و `assets/`
و `_redirects` وملفات الأيقونات. هذا المجلد هو ما سترفعه.

---

## الطريقة 1: Netlify بالسحب والإفلات (الأسهل — بدون أوامر)

1. افتح الموقع: <https://app.netlify.com/drop>
2. سجّل دخول مجاني (بحساب GitHub أو بريد إلكتروني).
3. اسحب مجلد **`dist`** كاملاً وأفلته في المنطقة المخصّصة في الصفحة.
4. انتظر ثوانٍ، ستحصل على رابط مثل: `https://random-name.netlify.app`
5. (اختياري) من إعدادات الموقع: `Site settings → Change site name` لتغيير الاسم.

لتحديث الموقع لاحقاً: نفّذ `npm run build` ثم اسحب مجلد `dist` من جديد.

---

## الطريقة 2: Vercel (ربط بمستودع Git — تحديث تلقائي)

هذه أفضل طريقة إذا كنت تستخدم GitHub، فكل تعديل يُنشر تلقائياً.

1. ارفع المشروع إلى مستودع على GitHub (انظر قسم Git في الأسفل).
2. ادخل <https://vercel.com> وسجّل دخول بحساب GitHub.
3. اضغط **Add New → Project** ثم اختر المستودع.
4. سيتعرّف Vercel على Vite تلقائياً. تأكد من الإعدادات:
   - **Framework Preset:** `Vite`
   - **Build Command:** `npm run build`
   - **Output Directory:** `dist`
5. اضغط **Deploy**. ستحصل على رابط `https://اسم-المشروع.vercel.app`.

---

## الطريقة 3: Cloudflare Pages

1. ادخل <https://dash.cloudflare.com> → **Workers & Pages → Create → Pages**.
2. اربط حساب GitHub واختر المستودع.
3. الإعدادات:
   - **Build command:** `npm run build`
   - **Build output directory:** `dist`
4. اضغط **Save and Deploy**. الرابط: `https://اسم-المشروع.pages.dev`.

> Cloudflare و Netlify يقرآن ملف `_redirects` تلقائياً، لا حاجة لإعداد إضافي.

---

## الطريقة 4: GitHub Pages (تحتاج خطوة إضافية)

GitHub Pages ينشر تحت مسار فرعي مثل `username.github.io/repo`، لذا يجب ضبط
`base` في إعدادات Vite أولاً وإلا لن تظهر الأنماط والملفات.

1. عدّل `vite.config.ts` وأضف اسم المستودع كـ base (استبدل `repo-name`):

```ts
export default defineConfig({
  base: "/repo-name/",
  plugins: [react(), tailwindcss()],
});
```

2. نفّذ `npm run build`.
3. ارفع محتوى مجلد `dist` إلى فرع `gh-pages`، أو استخدم إجراء GitHub Actions.
4. من `Settings → Pages` اختر الفرع المصدر.

> ملاحظة: لأن الموقع SPA، روابط التحديث المباشرة قد لا تعمل على GitHub Pages
> بدون حيلة إضافية (نسخ `index.html` إلى `404.html`). لذلك نوصي بـ Netlify أو
> Vercel لتجربة أسلس.

---

## رفع المشروع إلى GitHub (لازم للطرق 2 و 3 و 4)

```bash
git init
git add .
git commit -m "إطلاق موقع هياكل البيانات"
git branch -M main
git remote add origin https://github.com/USERNAME/REPO.git
git push -u origin main
```

تأكد من وجود ملف `.gitignore` يستثني `node_modules` و `dist` قبل الرفع.

## توصيتنا

- **تريد أسرع طريقة بدون Git؟** استخدم **Netlify Drop** (الطريقة 1).
- **تريد تحديثاً تلقائياً مع كل تعديل؟** استخدم **Vercel** (الطريقة 2).
